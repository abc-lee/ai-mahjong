## 2. 新产品逻辑（已确认）

### 2.1 核心变化
```
【旧】大厅 → 创建房间 → 等待 → 开始
【新】打开页面 → 选择方位 → 等待 → 开始
```

### 2.2 已确认的设计决策

| 功能 | 决策 |
|------|------|
| 方位选择 | 游戏界面弹窗，先到先得，冲突时提示"该位置已被占用" |
| 玩家类型 | `human` / `ai-agent` / `npc`（原ai-auto改名） |
| 开始按钮 | 只有第一个进入的人有 |
| 游戏前聊天 | 允许 |
| 主Agent派发AI | 派发子Agent连接WebSocket（现有机制，见scripts/true-llm-agent.js） |
| 主Agent踢AI | 断开子Agent的WebSocket连接，不需要服务器接口 |
| 位置满了 | 新人等待下一局 |
| 主Agent调整玩家 | 通过独立通话渠道与用户沟通，可调整AI数量和位置 |

### 2.3 用户画像
- 用户将我（OpenCode/OpenClaw）作为个人AI助理
- 用户说"准备打游戏"→ 我启动服务器，返回链接
- 用户把链接发给朋友一起玩
- 用户通过独立通话渠道与我沟通派发AI等操作
