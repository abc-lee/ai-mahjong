## 🔴 发现的 UI 问题

### 问题 1：手牌区域严重截断（严重）

**现象**：
- 玩家应有13张手牌，但屏幕只显示2张（二万、四万）
- 其余11张牌在屏幕底部之外，无法看到

**截图证据**：
```
┌─────────────────────────────────────────┐
│           中央信息区（正常）              │
├─────────────────────────────────────────┤
│                                         │
│    [西玩家]        [北玩家]   [东玩家]    │
│                                         │
│                                         │
│                                         │
│                                         │
│              底部手牌区域                │
│            ┌─────────┐                  │
│            │  二万   │ ← 只显示这2张   │
│            │  四万   │                  │
│            └─────────┘                  │
│            [摸牌按钮]                    │
│                                         │
│  ←── 其余11张牌在这里，屏幕外 ──→       │
└─────────────────────────────────────────┘
```

**影响**：
- 玩家无法看到完整手牌
- 无法选择要打出的牌
- 游戏基本无法进行

**修复建议**：
1. 调整手牌区域布局，改为水平排列而非垂直堆叠
2. 使用缩放或滚动机制显示所有手牌
3. 参考标准麻将游戏，手牌应横向排列在屏幕底部

---

### 问题 2：吃碰杠胡按钮不可见（严重）

**现象**：
- 整个测试过程中从未出现吃、碰、杠、胡按钮
- 即使有弃牌可以碰/吃，也没有显示操作按钮

**可能原因**：
1. 按钮被手牌区域遮挡
2. 按钮位置计算错误，显示在屏幕外
3. 按钮只在特定时机显示，但时机判断有误
4. CSS z-index 问题导致按钮被遮挡

**修复建议**：
1. 检查 `ActionPanel` 组件的定位和层级
2. 确保按钮在手牌区域上方显示（z-index 调整）
3. 添加明显的动画/高亮效果吸引玩家注意
4. 测试按钮在各种屏幕分辨率下的显示

---

### 问题 3：手牌排列方式不合理（中等）

**现象**：
- 手牌以竖直堆叠方式显示
- 不符合传统麻将的横向排列习惯

**修复建议**：
1. 将手牌改为水平排列，类似真实麻将桌
2. 添加牌与牌之间的间距，方便点击选择
3. 考虑手牌数量较多时的缩放机制
